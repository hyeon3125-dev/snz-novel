## 1. 이 게임이 돌아가는 방식 (그림 한 장)

```
원고 (마크다운 텍스트 파일)
   │
   ▼  변환기 (한 글자라도 바뀌면 멈추는 검증기 포함)
각본 데이터 (script.js — 거대한 목록일 뿐, 코드 아님)
   │
   ▼  브라우저 엔진 (5개의 작은 파일)
화면에 한 줄씩, 연출과 함께
```

핵심은 두 가지입니다.

- **AI가 없습니다.** 화면의 모든 문장은 작가가 쓴 그대로입니다. 게임은 "어떤 문장을, 어떤 순서로, 어떤 연출과 함께 보여줄까"만 결정합니다.
- **서버가 없습니다.** 그래서 운영비가 0원이고, 인터넷이 끊겨도 돌아가고, 10년 뒤에도 열립니다.

## 2. 내 소설로 만들고 싶다 (작가용)

### 준비물

| | 비용 |
|---|---|
| 내가 쓴 소설 (텍스트면 됨) | — |
| [GitHub](https://github.com) 계정 | 무료 |
| 컴퓨터 + Python ([python.org](https://www.python.org/downloads/)에서 설치, 5분) | 무료 |

### 따라하기 — 5단계

**① Fork** — 이 레포 우측 상단의 `Fork` 버튼을 누르면 내 계정에 복사본이 생깁니다. 이제 그건 당신 것입니다.

**② 원고를 형식에 맞추기** — 규칙은 딱 3개입니다:

```markdown
## Vol.1 Ch.1 — 시작

첫 문단입니다. 빈 줄로 구분된 한 덩어리가 화면에 한 번에 나옵니다.

탭 한 번에 이 문단이 나옵니다.

-----

하이픈 다섯 개는 씬(장면)의 경계입니다. 화면에 여백이 생깁니다.

## Vol.1 Ch.2 — 다음 장

챕터 제목은 화면에 카드로 표시됩니다.
```

| 규칙 | 의미 |
|---|---|
| 빈 줄로 구분된 문단 | 탭 1회 = 1문단 |
| `-----` (하이픈 5개) | 씬 경계 |
| `## Vol.권 Ch.화 — 제목` | 챕터 시작 |

**③ 원고 교체** — `manuscript/` 폴더의 파일을 지우고 내 원고 파일을 넣습니다.

**④ 변환** — 터미널(명령 프롬프트)에서 아래를 복사해 붙여넣습니다:

```bash
cd tools
python3 parse_snz.py ../manuscript/내원고.md -o ../game/script.js -a annotations/all.json
python3 verify_integrity.py ../game/script.js ../manuscript/내원고.md -a annotations/all.json
```

두 번째 줄이 검증기입니다 — 변환 과정에서 원고와 **한 글자라도** 달라졌으면 멈추고 알려줍니다. `무결성 100%`가 나오면 끝.
(처음엔 `annotations/all.json`의 내용을 `{}` 빈 괄호 두 개로 비우고 시작하세요 — 이 파일은 이 작품 전용 연출 설정입니다.)

**⑤ 공개** — 바꾼 파일을 GitHub에 올리고(웹에서 업로드 버튼으로도 됩니다), 레포 Settings → Pages → Source를 `main` / `(root)`로 켜면, 몇 분 뒤 `내아이디.github.io/레포이름` 이 **내 게임의 공개 링크**가 됩니다. 호스팅비 0원.

### 연출 입히기 (선택 — 나중에 해도 됨)

연출은 원고를 건드리지 않고 별도 파일(`tools/annotations/all.json`)에 적습니다. "몇 번째 씬의 몇 번째 줄에 어떤 효과"를 사전처럼 등록하는 방식입니다:

```json
{
  "fx": {
    "v01_c001_s02": { "3": "echo" }
  }
}
```

→ "1권 1챕터 두 번째 씬의 4번째 줄(0부터 셈)에 `echo` 효과."

| 효과 | 화면에서 일어나는 일 |
|---|---|
| `echo` | 직전 문장이 잔상처럼 한 번 어른거림 |
| `loss` | 출력됐던 문장이 단어 단위로 사라짐 — 스크롤을 올려도 없음 |
| `blank` | 문장 가운데 단어 하나가 빈칸으로 나옴 |
| `crack` | 화면 가장자리에 균열이 하나 늘어남 (끝까지 회복 안 됨) |
| `shake_s` / `shake_m` | 화면 흔들림 약/중 (폰은 진동) |
| `seat` | 0.8초 완전 정지 — 모든 UI가 사라지고 무음 |
| `light` | 배경 색이 30초에 걸쳐 천천히 따뜻해짐 |
| `pause_b` | 1.2초간 탭이 안 먹힘 — 강제 호흡 |

독자의 행동을 요구하는 **인터랙션**은 씬 단위로 겁니다:

| 타입 | 독자가 해야 하는 것 |
|---|---|
| `hold` | 화면을 누르고 있어야 진행 (손을 대고 있는 것) |
| `release` | 누르고 있던 것을 놓아야 진행 |
| `silence` | N초간 아무것도 하지 않아야 진행 (침묵이 답인 장면) |
| `timeout_choice` | 제한시간 안에 선택 — 안 고르면 "고르지 않았다"가 기록됨 |
| `shake` | 기기를 흔들어야 진행 |
| `trace` | 화면의 흐린 자국을 손가락으로 따라 그어야 보임 |

**한 가지 조언**: 효과는 아끼세요. 이 작품은 1만 1천 줄에 효과를 16곳에만 걸었습니다. 글에 근거가 있는 자리에만 연출을 넣으면, 들어간 자리가 정말로 울립니다.

## 3. 이런 아키텍처를 만들고 싶다 (개발자 지망용)

프레임워크도 빌드 도구도 없이, 바닐라 JS 파일 5개로 만들어졌습니다. 그래서 **읽으면 전부 이해할 수 있습니다.**

### 핵심 아이디어 5개

1. **콘텐츠는 데이터, 엔진은 코드** — 소설 전체가 `script.js`라는 하나의 자료구조입니다. 엔진은 그걸 한 줄씩 꺼내 보여줄 뿐. 콘텐츠를 바꿔도 엔진은 그대로입니다.
2. **결정론** — 난수 없음, AI 없음. 같은 입력은 언제나 같은 결과. 그래서 기계로 검증할 수 있습니다 (이 레포는 매 푸시마다 전체 완주를 자동으로 돌립니다).
3. **사이드카 어노테이션** — 원본(원고)을 절대 고치지 않고, 메타데이터를 옆 파일에 둡니다. 원본의 무결성을 기계가 보증할 수 있게 됩니다.
4. **검증 게이트** — "원고와 한 글자도 다르지 않은가"(verify_integrity), "모든 씬에 도달 가능한가"(validate_graph), "완주가 되는가"(smoke). 게이트를 통과 못 하면 출고가 안 됩니다.
5. **의존성 0** — npm install이 없으면 깨질 것도 없습니다. 정적 파일은 가장 오래 사는 배포 형태입니다.

### 코드 읽는 순서 (전부 합쳐 ~1,100줄)

```
1. game/script.js 의 맨 앞부분  — 데이터 구조: { order: [...], scenes: {...}, units: {...} }
2. game/director.js 의 step()   — 엔진의 심장. "다음에 보여줄 것"을 결정하는 함수 하나
3. game/main.js 의 advance()    — 탭 1회 = step()을 보이는 것이 나올 때까지 반복
4. game/stage.js                — 결정된 것을 화면에 그리는 일 전부
5. game/input.js                — 탭·제스처를 이벤트로 정규화
```

### 미니 엔진 — 개념은 이게 전부입니다

```js
const scenes = { s1: { lines: ["첫 줄.", "둘째 줄."], next: "s2" }, s2: { ... } };
let cur = "s1", idx = 0;

document.onclick = () => {
  const sc = scenes[cur];
  if (idx < sc.lines.length) {
    show(sc.lines[idx++]);            // 다음 줄
  } else if (sc.next) {
    cur = sc.next; idx = 0;           // 다음 씬
  }
  save(cur, idx);                     // localStorage — 이게 세이브의 전부
};
```

여기에 ① 연출 태그를 줄에 붙이고 ② 씬에 인터랙션 조건을 걸고 ③ "본 것"을 기록해 뒷장면의 연출을 바꾸면(회수 게이트) — 이 레포의 엔진이 됩니다.

### 더 가보기

- **회수 게이트**: 복선 장면을 통과하면 `seeds["복선이름"]=true`, 회수 장면에서 그 값이 있으면 연출 풀버전 / 없으면 묵음. "본 사람에게만 울리는" 구조.
- **도달 상태**: 엔딩을 갈래로 만들지 않고, "어떻게 읽었는가"(복선 회수율, 회피한 선택)에 따라 결말부의 개방 범위가 달라지게.
- **시점별 텍스트 문법**: 색을 바꾸지 말고 텍스트가 화면에 놓이는 방식을 바꿔보세요 — 타임스탬프가 붙는 시점, 렌즈 안에만 보이는 시점, 글자가 떠다니는 시점.

## 4. 자주 묻는 질문

**Q. 폰에서 되나요?** — 됩니다. 진동·흔들기 인터랙션은 폰에서 더 좋습니다.

**Q. 돈이 드는 단계가 하나라도 있나요?** — 없습니다. GitHub 계정, Pages 호스팅, Python, 이 엔진 — 전부 무료입니다.

**Q. 이 엔진으로 만든 내 작품의 권리는?** — 엔진 코드는 MIT라 자유롭게 쓰면 되고, **당신의 원고는 당신 것입니다.** 라이선스도 당신이 정합니다. (이 레포의 소설 본문만 CC BY-NC-SA 4.0입니다 — 그건 이 작품의 조건이지 엔진의 조건이 아닙니다.)

**Q. 원고 형식이 좀 다른데요 (씬 구분자가 다르다든가).** — `tools/parse_snz.py` 상단의 정규식 몇 줄이 형식 정의의 전부입니다. 거기를 고치면 됩니다. 어렵게 느껴지면 Issues에 질문을 남겨주세요.

**Q. 막혔어요.** — [Issues](https://github.com/hyeon3125-dev/snz-novel/issues)에 한국어로 편하게 남겨주세요.
